Bellhop 水声射线追踪工具：从零到实战

1. Bellhop 是什么？

Bellhop 是 Acoustics Toolbox 中的水声传播建模工具，采用高斯波束追踪法在 2D 垂直剖面（距离-深度）上计算水下声传播。它是水声学领域最广泛使用的射线追踪工具之一，由 Michael B. Porter 开发，适用于频率 >100Hz 的深海和浅海场景。

核心能力：

  • 声线轨迹计算 (compute_rays) — 可视化声传播路径
  • 到达时间与复幅度 (compute_arrivals) — 信道冲激响应建模
  • 传播损失 (compute_transmission_loss) — 声场能量分布
  • 本征声线 (compute_eigenrays) — 连接特定收发对的声线

适用场景：深海远程声传播、声纳性能评估、水声通信信道建模、探通一体化链路仿真。

3. 下载与编译

3.1 获取源码

官方源 oalib.hlsresearch.com 在国内网络环境下下载不稳定（超时、断开）。推荐使用 GitHub 镜像 A-New-BellHope/bellhop，这是 Scripps 海洋研究所维护的修复版，修复了原版多个数值 bug。

教训：不要从 oalib.hlsresearch.com 直接下载（国内超时严重），GitHub 镜像更稳定。

3.2 编译

Makefile 已预设 gfortran 编译选项（-O2 -ffast-math -march=native），一般无需修改。编译产物在 Bellhop/bellhop.exe（约 27MB）。

错误 后果 正确做法
频率行格式不对 Bellhop 静默退出，无 .arr 输出 频率独占一行

8. 实战经验与陷阱

8.1 网络下载

  • ❌ oalib.hlsresearch.com 直连不稳定 → ✅ GitHub 镜像 A-New-BellHope/bellhop

8.2 .env 格式

  • Bellhop 对格式极度敏感，错一个字段就静默失败（exit code 0，不报错，不生成有效输出）
  • 务必检查 .prt 文件确认参数被正确解析
  • 距离单位是 km，不是 m

8.5 多径数量选择

Bellhop 可能返回几十条到达路径。用于链路级仿真时：

  • 太少（<4 条）：丢失重要多径分量，BER 过于乐观
  • 太多（>16 条）：弱路径引入 ISI，BER 反而恶化
  • 推荐：保留 4-8 条最强路径，按 |amplitude| 降序选取

8.6 性能考虑

  • compute_rays(env, nbeams=800)：~0.3s
  • compute_arrivals(env)：~0.05s
  • compute_transmission_loss(env, 501×1001 grid)：~30-60s（取决于距离和波束数）
  • 同一 env 参数下信道抽头可缓存复用，避免重复调用 Bellhop

10. 总结

环节 要点
获取 GitHub A-New-BellHope/bellhop（不要用官方 FTP）
编译 gfortran + make all，产物 27MB
安装 复制到 ~/at/bin/bellhop，ln -s 到 PATH 中的 bellhop.exe
Python pip install arlpy，通过 pm.compute_*() 调用
关键陷阱 .env 格式严格、距离单位 km、复数幅度 vs 幅度+相位、功率归一化、相对时延
最佳实践 缓存信道抽头、限制 4-8 条最强路径、检查 .prt 文件、用 matplotlib 不用 Bokeh

Bellhop 是水声信道建模仿真的基础工具，掌握其安装配置是后续所有水声项目（探通一体化、DOA 估计、声纳性能评估）的前提。建议在项目启动阶段就完成 Bellhop 环境搭建和验证。

本文基于 DSSS-MSK 探通一体化项目的实战经验。环境：WSL2 + Ubuntu 22.04 + gfortran 11.4.0 + Python 3.10 + arlpy 1.9.1。
